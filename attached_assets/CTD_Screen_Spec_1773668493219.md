# CTD Folders - Screen Specification

This document provides a detailed breakdown of the CTD / eCTD Folder Structure master screen.

## 1. Top Summary Cards
To align with the design system, the CTD screen should include 4 dynamic summary cards:
1. **Total Modules**: Count of all folders/modules in the system.
2. **Root Modules**: Count of all Level 1 top parent modules.
3. **Active Folders**: Count of all active folders.
4. **Inactive Folders**: Count of all inactive folders.

## 2. Grid View & Filters (MemoizedDataGridComponent)
- **Data Display**: A flat list or hierarchical drill-down representing the CTD folder numbering (e.g., `Module 1`, `Module 1.2`, `1.2.3`).
- **Standard Controls**: 
  - Search Box.
  - Page Length Dropdown.
  - Refresh Icon.
- **Top Add Button**: A prominent "Create Root Module" or "Create Folder" button.

## 3. Create / Edit Form Controls

When launching the Add or Edit form ("Create Root Module" UI), the layout should be full-screen consistent with the project's design.

### Form Field Bindings
| Control | Type | Dependency / Logic | Validation |
| :--- | :--- | :--- | :--- |
| **Parent Folder (Optional)** | Dropdown | Allows nesting. If left as `No Parent (Root)`, it creates a top-level module (e.g., Module 1). | Optional |
| **Folder Code** | Textbox | The shortcode identifier (e.g., `Module 1.2` or `1.1.1`). | Required |
| **Folder Name** | Textbox | The long descriptive name of that specific CTD section. | Required |
| **Sort Order** | Number Box | Controls visual rendering order in trees. | Optional |
| **Description** | Textarea | Notes about the module. | Optional |

## 4. CRUD Operations & Validations
- **Duplicate Prevention**: The system must validate that the identical **Folder Code** does not already exist horizontally under the same Parent Folder.
- **Edit**: Clicking the single Checkbox + Pen Icon loads the specific folder details into the form.
- **Delete**: Clicking Checkbox(es) + Trash Icon. The system must **block deletion** if that CTD Folder is actively linked/mapped to an existing Category or Document to prevent orphaned data.

### Selection Logic Helper
```tsx
const _onItemSelected = (item: any): void => {
    if (item.length > 0) {
        if (item.length == 1) {
            setisSelectedData(true);
            setUpdateItem(item);
            setIsDisplayEditButtonview(true); // Show Edit
            setDeleteId(item[0].ID);
        } else {
            setisSelectedData(true);
            setUpdateItem(item);
            setIsDisplayEditButtonview(false); // Hide Edit for multiple
        }
        setisDisplayEDbtn(true); // Show Delete
    } else {
        setisSelectedData(false);
        setUpdateItem([]);
        setDeleteId(0);
        setisDisplayEDbtn(false); // Hide both
    }
};
```
